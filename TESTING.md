# Q/GDW 10376.2 解析器 — 覆盖整理与测试说明

> **协议依据**：`doc/17 用电信息采集系统通信协议 第2部分：集中器本地通信模块接口.pdf`（**Q/GDW 10376.2—2019**）。  
> **实现**：`GDW1376_2/` 编译为 `GDW1376_2_64.dll` 等；对外解析入口见 `GDW1376_2/inc/GDW1376_2.h`。  
> **规格与缺陷登记**：`SPEC.md`。本文专注 **「解析覆盖缺口」** 与 **「如何测」**。

## 1. 解析覆盖整理（相对 2019 正文第 6.5 节）

### 1.1 标准已定义、此前缺分支（已补）

| 项 | 说明 | 实现 |
|----|------|------|
| **AFN=05H，F200** | 「控制拒绝节点上报」（表 38 / 表 48） | `AFN05.c`：`AFN05_FN200` + `case 200`（2026-05 起） |

### 1.2 标准性质特殊、不宜强解字段

| 项 | 说明 | 建议 |
|----|------|------|
| **AFN=F0H** | 内部调试，**厂家自定义**（6.5.14） | **已实现**：`AFNf0.c` 输出 Fn 推算值 + **数据域 HEX 转储**；不做字段级语义 |
| **AFN=10H，F104** | 表 58 列出名称，**6.5.8.3 / 6.5.8.5 无 F104 上行专用分表** | 维持「常见长度启发式 + 缺表说明 + 转储」；若企标/增补文档有定义，再以单独 fixture 对齐 |

### 1.3 已有分支、但与 PDF 表仍可能不一致（实现债，非「无 case」）

| 项 | 说明 |
|----|------|
| **AFN=14H，F4** | **已按 2019 版修正**：上行 **表 113**（固定 5 字节）；下行 **表 117**（5+N 变长 + HEX 内容）；若与现网帧不一致再补 fixture |
| **AFN=05H 部分 Fn** | 与表 41（F3 控制字）、表 44（F6 禁止/允许）、表 46/47（F100/F101）等字段名与枚举对齐（见 `SPEC.md` P1 项） |
| **AFN=04H 等** | **F1/F2/F3 已补** 下行长度校验（见 `SPEC.md` P1-07） |
| **多 Fn 同帧** | `get_clear_fn` 仅处理一个 Fn；多 bit 时需产品策略：循环解析或显式报错（见 `SPEC.md` §2.3） |

### 1.4 `AFN.c` 已接线的应用层 AFN

`print_AFN` 当前分发：**00、01、02、03、04、05、06、10、11、12、13、14、15、F0、F1**。与 2019 版 **6.5 主文**范围一致；若存在**文档外的扩展 AFN**，将返回 `-ERR_AFN_FN`。

### 1.5 非本库必测范围

- 各表中 **F2～F248「备用」**：不要求实现解析。  
- **DL/T 645、DL/T 698.45** 等嵌套负载：产品目标一般为 **L + HEX 展示**，不做完整电表应用层解析（与 `SPEC.md` §1.2 一致）。

## 2. 测试目标与原则

| 维度 | 目标 |
|------|------|
| **协议** | 字段含义、字节偏移、变长公式、边界与 **PDF 数据单元表**一致 |
| **工程** | 不崩溃；**错误码稳定**（负 `ERR_*`，与 `GDW1376_2_error()` 一致）；输出可做回归对比 |
| **衔接** | 每条重要修复（尤其 P0）尽量带 **至少 1 条金样报文**，避免回归 |

## 3. 测试分层（推荐）

### 3.1 链路层与入口

| 场景 | 期望（示例） |
|------|----------------|
| 合法单帧 | `ret == 0`，输出含关键字段或成功路径文案 |
| **粘包 / 超长**（合法帧后多字节） | `ret != 0`；新构建 DLL 上 **`ERR_CHKFRM_LEN_LONG` 为负值**（见 `SPEC.md` P0-02） |
| 坏帧（错 CS、缺 68/16、长度非法） | 对应 `ERR_CHKFRM_*` / `ERR_INPUT` 等，且不崩溃 |

**现成脚本**：`scripts/smoke_m0.py`（加载 `GDW1376_2/GDW1376_2_64.dll`）：**AFN03 金样**、**粘包**、**AFN05-F200 下行**、**AFN=F0H 转储**（组帧 CS 与 `chkfrm` 一致）。

### 3.2 应用层分发

| 场景 | 期望 |
|------|------|
| 未知 AFN | `-ERR_AFN_FN`（或规格书规定的统一错误） |
| 已知 AFN、未实现 / 未支持的 Fn | `-ERR_AFN_FN` 与明确提示 |
| **AFN05-F200**（实现后） | 专用用例进入新分支，且长度与表 48 一致 |

### 3.3 数据单元（金样 + 边界）

- **金样（Golden）**：固定十六进制串 + 断言输出**子串**（或结构化期望表），与「按 PDF 手算」一致。  
- **边界**：`应用层长度 = 表要求−1 / 恰满足 / +1`；变长项 **n=0、n=1、允许范围内的较大 n**。  
- **字节序**：对 **BIN 2** 的序号、长度等用已知数值区分低字节在前是否实现正确。  
- **多记录 Fn**（如 AFN06-F4、AFN10 部分查询）：至少 **2 条记录** 的样例，防止循环/步长错误。

### 3.4 回归与 CI（建议门禁）

1. `make -C GDW1376_2 dll64`（或团队约定的 32/64 目标）  
2. `python scripts/smoke_m0.py`（可扩展为下文「fixture 驱动」脚本）  
3. 可选：对解析入口做**轻量 fuzz**（随机 HEX、有上限长度），主要防越界与死循环  

### 3.5 py103762 解析界面（仅 Python 版，不改 Qt）

- **DLL 加载顺序**：`py103762/../GDW1376_2/GDW1376_2_64.dll` → 同目录 `GDW1376_2.dll` → `py103762` 目录下 64/32 位 DLL；便于开发时直接用仓库内编译产物。  
- **缩进风格**（窗口底部 **「缩进」** 下拉）：**原始** / **块分隔**（分隔线、解析成功/失败前自动空行）/ **树状竖线**（正文行前加 `│`）；为 **Python 侧后处理**，不改变 C 库输出。  
- **HTML 安全**：解析结果经 **`html.escape`** 后放入 `<pre style="white-space:pre-wrap">`，避免 `<>&` 破坏显示。  
- **字符编码**：DLL 返回字节串依次尝试 **GBK、UTF-8** 解码。

## 4. 金样与目录约定（建议）

| 约定 | 说明 |
|------|------|
| **目录** | `scripts/fixtures/`（已建示例与说明：`scripts/fixtures/README.md`）或 `tests/fixtures/`（二选一统一即可） |
| **命名** | `afn{XX}_fn{YYY}_{brief}.hex` 或 `.txt`，例如 `afn03_fn01_vendor.hex`；**一行或可带空格**的十六进制，与 `GDW1376_2_parse_foy_py` 输入一致 |
| **元数据** | 同名的 `.md` / `.json` 可选：说明来源（脱敏现网 / 手算组帧）、期望子串列表、`ret` 期望 |
| **CS 计算** | 与实现一致：自 **控制域 C** 起至 **CS 前一字节** 的算术和（溢出丢弃），组金样时需自行算对 |

**金样来源**：脱敏现网抓包；或严格按 PDF 表手写数据单元再组完整帧。

## 5. 扩展 `smoke_m0.py` 的方向（可选实现）

将单脚本扩展为 **数据驱动**，例如：

- 多组：`hex`、`expect_substrings`（列表）、`forbidden_substrings`（可选）、`ret`（可选，默认 0）  
- 失败时打印 DLL 路径、`ret`、`GDW1376_2_error(ret)`、输出前若干字符  

便于为 **AFN05-F200、AFN14-F4、AFN10-F104** 等逐条追加用例而少改代码结构。
